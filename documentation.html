Data Sources

The datasets used in this project were obtained from trustable and publicly accessible government repositories related to Australian road safety and transport statistics. The primary data sources are as below.

Mobile Phone Enforcement Data — Australian Road Safety Agencies

Core dataset contains road safety enforcement activities undertaken across Australian states and territories from 2008 to 2024. It includes enforcement data from 8 agencies as below:

  • NSW Police Force
  • Victoria Police
  • Queensland Police Service and Department of Transports and Main Roads
  • South Australia Police Force
  • Western Australia Police Force
  • Tasmania Department of Police and Emergency Management
  • North Territory Police Force
  • Australian Federal Police and Access Canberra

The dataset is available for download in Mobile Phone Use Fines.

Australian Infrastructure and Transport Statistics Yearbook 2024

The Yearbook provides a comprehensive annual compilation of infrastructure and transport statistics for use by policymakers, industry professionals, transport analysts, and the broader Australian community. For this project, we specifically extracted driver licence data from Table 6.13a–i of Chapter 6, which details the licence holders by age and gender across Australian states and territories. This dataset serves as a key input for analysing trends in traffic enforcement relative to the number of licensed drivers.

The dataset is available for download in Road Statistics Yearbook 2024.

Processed Datasets

All datasets were cleaned, normalised, and formatted using KNIME to ensure consistency across states and enforcement methods. These processed files were used directly in the dashboard visualisations.

  • Annual Fines for Mobile Phone Use (CSV)
  • Annual Fines by Jurisdiction (CSV)
  • Annual Fines by Jurisdiction (with Detection Method Filter) (CSV)
  • Fines per 10,000 Driver Licences by Jurisdiction per year (CSV)
  • Fines per 10,000 Driver Licences by Jurisdiction per year (with Detection Method Filter) (CSV)

Data Cleaning

Below are the clean and preprocessing steps for the datasets.

Dataset 1: Mobile Phone Offences

  1. Loading the Dataset
    Loaded the first dataset using the KNIME CSV Reader node (mobile phone offences data).
  2. Removing Unused Columns
    Removed unneeded columns (start date, end date, location, age group) using the Column Filter node.
  3. Filtering for Mobile Phone Offences
    From 12,179 records containing various offences (speeding, seatbelt, etc.), kept only rows for the mobile phone use category using the Row Filter node, resulting in 2,517 records.
  4. Removing Unknown Detection Methods
    Removed rows where detection method was Unknown using another Row Filter node, leaving 2,435 clean records.
  5. Sorting the Data
    Sorted in ascending order by year and jurisdiction using the Sorter node for readability.
  6. Grouping Detection Methods
    Simplified detection methods into two categories using the Rule Engine node: Police-issued fines and Camera-issued fines, enabling clear trend comparisons.

Dataset 2: Licence Holders

  1. Loading the Dataset
    Loaded the licence holder dataset via the Excel Reader node. The tabular format required additional cleaning compared to the first dataset.
  2. Removing Unused Columns
    Used the Column Filter node to remove columns not required for analysis.
  3. Handling Missing Rows
    Applied the Row Filter node to remove rows with missing or empty values for completeness.
  4. Creating and Cleaning the Jurisdiction Column
    Replaced the original jurisdiction indicator (Table 6.13a–i) using the Rule Engine node, then used the Missing Value node to fill missing jurisdiction values for previous rows.
  5. Cleaning the Gender Column
    Filled missing gender values using the same approach to ensure consistency.
  6. Formatting the Year Column
    Converted full dates (e.g., 31 June 2022) to year-only format (e.g., 2022) using the String Manipulation node, and changed the Year column type from categorical to numerical for visualisation.
  7. Renaming and Sorting Columns
    Renamed the Year and Total Licences columns for clarity and arranged their order with the Sorter node to improve structure and readability.

Glossary

Terms and definitions used across the dashboard and documentation.

Jurisdiction

State or territory in Australia.

Detection Method

Police-issued vs camera-detected enforcement.

Year

Time period for enforcement data aggregation.

Fine rate per 10,000 driver licences

Fair comparison metric across jurisdictions with different population sizes.
